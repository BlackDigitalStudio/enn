# Обсуждение физического движка

Сегодня обсуждали архитектуру физического движка для нашего проекта.

## Иерархия классов

Базовый класс `PhysicsBody` отвечает за хранение массы, позиции и скорости.
От него наследуется `RigidBody`, который добавляет радиус и метод `checkCollision`.

Класс PhysicsWorld управляет всей симуляцией. Его метод `step` на каждом тике:
1. Применяет гравитацию через `applyForce`
2. Вызывает `update` для каждого тела
3. Запускает `resolveCollisions` для обработки столкновений

## Проблема производительности

При большом количестве тел `resolveCollisions` работает за O(n^2).
Нужно внедрить spatial partitioning (QuadTree или Grid) в PhysicsWorld.

## Решение

Договорились что RigidBody::checkCollision останется как есть,
а оптимизацию broad-phase сделаем отдельным классом CollisionDetector.
